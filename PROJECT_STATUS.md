# Project status — технический и бизнес‑отчёт

Дата: 2026-02-19

## Кратко (one‑line)
Проект — e‑commerce (табачные товары) на Next.js + Supabase: полнофункциональный каталог, админ‑панель (CRUD товаров/категорий/заказов), автоматический сбор ошибок и удобный UX; основные части реализованы и готовы к деплою, есть админский интерфейс для мониторинга ошибок.

---

## Бизнес‑составляющая
- Целевая функциональность: интернет‑магазин табачных изделий для Казахстана (каталог → корзина → заказ, админ‑управление).  
- Пользовательские потоки реализованы: просмотр каталога, фильтры, карточки товара, оформление заказа (заказы сохраняются в БД).  
- Админ‑поток: управление товарами, категориями, заказами, просмотр отчётов об ошибках.
- Риски / соответствие: контент для 18+ — учтено в бизнес-описании (age check).  

Бизнес‑рекомендации / что добавить дальше
- Интеграция платёжного шлюза / возвраты / диспуты (высокий приоритет).  
- Аналитика конверсий и A/B; email‑маркетинг (следующий этап).  
- SLA/резервирование базы при росте (Supabase Pro → Business).  

---

## Технический стек
- Frontend: Next.js (App Router), TypeScript, Tailwind CSS, React.  
- Backend: Supabase (Postgres, Auth, Storage, Edge Functions возможны).  
- Hosting: фронтенд можно размещать на Vercel/3hcloud (VPS/docker).  
- Monitoring/Errors: самописный ErrorBoundary + `error_reports` в Supabase.  

---

## Что реализовано сейчас (технически)
- UI / UX
  - Красивый каталог с карточками категорий (`src/app/catalog/page.tsx`).
  - Sidebar + мобильные фильтры, настраиваемая сетка карточек.
- Категории
  - Иерархические категории + изображения, админ CRUD (`/admin/categories`).
  - Drag & drop и кнопки «вверх/вниз» для сортировки (`sort_order`) с сохранением в БД.
  - Header теперь использует динамические категории (из БД) вместо хардкода.
- Товары
  - Админский CRUD товаров; категория для товара — иерархия поддержана (`src/app/admin/products/**`).
  - `useProducts` дополнен фильтрацией по `categoryIds`.
- Заказы
  - Страница админа для управления заказами (фильтры/статусы/обновление статуса).
- Ошибки / мониторинг
  - `ErrorBoundary` (клиент) автоматически отправляет отчёты и предоставляет кнопку «Отправить разработчикам».
  - API route `POST /api/report-error` сохраняет в таблицу `error_reports` (сервисный ключ на сервере).
  - Админ‑страница для `error_reports` с фильтрами, просмотром и отметкой `resolved` (`/admin/error-reports`).
- БД / миграции
  - Миграции: `013_categories_update.sql`, `014_error_reports.sql`, `015_error_reports_add_resolved.sql` и др.

Файлы и точки входа (важное)
- `src/app/catalog/page.tsx` — основной каталог UI
- `src/app/admin/categories/page.tsx` — админ: категории (drag&drop)
- `src/app/admin/error-reports/page.tsx` — админ: отчёты об ошибках
- `src/components/ErrorBoundary.tsx` — клиентский ErrorBoundary
- `src/app/api/report-error/route.ts` — серверный API для ошибок
- `supabase/migrations/014_error_reports.sql`, `015_error_reports_add_resolved.sql`
